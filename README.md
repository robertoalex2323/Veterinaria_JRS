# FARMACEUTICO:
Semana 4:
Se inicia el proyecto. Se definen requerimientos del sistema (verificación de receta, ventas, inventario, etc.) y se diseña la estructura general del sistema.

Semana 5:
Se desarrolla el frontend base (interfaz principal del sistema: menú, navegación, pantallas iniciales).

Semana 6:
Se implementa el módulo de gestión de inventario (backend): registro de medicamentos, stock, entradas y salidas.

Semana 7:
Se desarrolla el frontend del inventario y conexión con el backend (visualización de medicamentos).

Semana 8:
Se implementa el módulo de verificación de receta veterinaria (backend): validación de datos, control de recetas.

Semana 9:
Se desarrolla el frontend de verificación de recetas (formulario, validación visual, alertas).

Semana 10:
Se implementa el módulo de dispensación de medicamentos (backend), enlazado con recetas e inventario.

Semana 11:
Se desarrolla el frontend de dispensación de medicamentos (selección de productos, confirmación).

Semana 12:
Se implementa el módulo de registro de ventas (backend): almacenamiento de transacciones.

Semana 13:
Se desarrolla el frontend de registro de ventas (historial, comprobantes).

Semana 14:
Se implementa el módulo de información sobre uso de medicamentos (backend): recomendaciones, instrucciones.

Semana 15:
Se desarrolla el frontend de información al cliente (pantalla de consejos y uso adecuado).

Semana 16:
Se realizan pruebas integrales del sistema (testing de todos los módulos).

Semana 17:
Se corrigen errores, optimiza rendimiento y se mejora la interfaz (UX/UI).

Semana 18:
Se realiza la implementación final y despliegue del sistema.
